# Contributing to Edwin

**Edwin is a tool for people who build things. If you find a problem, fix it. If you want a feature, build it.**

This project is built by someone who uses AI agents every day to do real work. The expectation is that you do too.

## The philosophy

You have an AI agent. Use it. Edwin exists because one person decided to build what they needed instead of waiting for someone else to build it. Contributions should come from the same energy.

## Discussions

Yes. Questions, ideas, architecture conversations, show-and-tell of what you built on top of Edwin -- all welcome. This is where the community lives.

## Bugs

If something is broken before you can get Edwin running (setup.sh fails, Docker won't start, dependency issues), open an Issue. Those are infrastructure problems and we want to know about them.

Once Edwin is running and you have your agent, the expectation changes. You have the same tools we used to build this. If you hit a bug:

1. Describe it clearly (what happened, what you expected, steps to reproduce)
2. Ask your agent to fix it
3. Submit the fix as a PR

Most bugs in Edwin are fixable by the same agent that runs Edwin. That's the point. We'll happily merge a clean fix from anyone.

## Feature requests

No. If you want a feature:

1. Open a Discussion to talk about the idea
2. Build it with your agent
3. Test it
4. Submit a PR

"Wouldn't it be cool if Edwin could..." is a Discussion. A working PR is a contribution. We accept contributions.

## Pull requests

- Solve a real problem you actually had
- Include tests or a clear test plan
- Don't break existing functionality
- Follow existing code patterns
- Build and test with an AI agent (we practice what we preach)
- Open a Discussion first if the change is significant

PRs without prior discussion on significant changes will be closed.

## What we're NOT

- A feature factory that takes orders
- A support desk that fixes your setup for you
- A project that accepts scope creep through wishful Issues

## What we ARE

A community of builders who use AI agents to solve their own problems and share the results. If that's you, welcome.
