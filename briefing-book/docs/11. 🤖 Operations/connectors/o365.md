# O365 Connector

Syncs Microsoft 365 data via the Microsoft Graph API -- email, calendar, Teams messages, SharePoint, and OneDrive.

## Quick Setup

Requires an Azure AD app registration with client credentials (application permissions) and optionally delegated auth for Teams chat.

### 1. Azure AD App Registration

Create an app registration in Azure Portal with these **application permissions**:
- `Mail.Read` (or `Mail.ReadBasic.All`)
- `Calendars.Read`
- `ChannelMessage.Read.All`
- `Sites.Read.All`
- `Files.Read.All`

For Teams chat (send/read DMs), you also need **delegated permissions**:
- `Chat.ReadWrite`
- `ChatMessage.Send`
- `Chat.Create`
- `Chat.ReadWrite.All`

### 2. Store Credentials

Create `~/.edwin/credentials/o365/env`:
```bash
AZURE_TENANT_ID=your-tenant-id
AZURE_CLIENT_ID=your-client-id
AZURE_CLIENT_SECRET=your-client-secret
EDWIN_EMAIL=your-email@company.com
```

Or set these as environment variables / in `~/Edwin/.env`.

### 3. Delegated Auth (for Teams chat)

Run the connector once -- it will open a browser for Microsoft login:
```bash
./connectors/o365/o365 teams-login
```

Token is cached at `~/.config/edwin/o365-delegated-token.json` with auto-refresh.

### 4. Verify

```bash
./connectors/o365/o365 status
```

## What It Captures

- **Mail**: Full email messages (subject, from, to, date, body, read status) -- one file per email
- **Calendar**: Events with title, time, location, attendees, description, Meet links
- **Teams**: Channel messages from all joined teams and channels
- **SharePoint**: Document library listings and site content
- **OneDrive**: File metadata from the user's OneDrive

## How It Works

- Uses client credentials flow for app-level access (mail, calendar, SharePoint, OneDrive)
- Uses delegated auth with refresh tokens for Teams chat operations
- Token cached at `~/.config/edwin/o365-app-token.json` (~1 hour lifetime, auto-refresh)
- Rate limiting with exponential backoff and Retry-After header support
- On 401: invalidates cached token and retries with fresh token
- Incremental sync via Graph API delta links and date tracking

## Output Format

```
~/Edwin/data/o365/
  mail/
    2026-04/
      2026-04-10-143022-a1b2c3d4.md    # One file per email
  calendar/
    2026-04/
      2026-04-10.md                     # Day-grouped events
  teams/
    {team-name}/
      {channel-name}/
        2026-04/
          2026-04-10.md
  sharepoint/
    {site-name}/
      ...
```

## Cadence

- Mail: every 15 minutes
- Calendar: every 30 minutes
- Teams: every 60 minutes
- SharePoint: every 60 minutes
